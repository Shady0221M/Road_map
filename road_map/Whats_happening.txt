3) 🔄 Flow of each functionality (end-to-end)
🔹 A. Load Learning Page (User)
UI (ClusterList / ChapterList)
    ↓
useContentData()
    ↓
GET /api/chapters/get?subject=...
    ↓
GET /api/chapters/concepts/get?chapterId=...
    ↓
UI renders clusters → chapters → concepts
🔹 B. Expand Chapter
User clicks ChapterTab
    ↓
fetch(`/api/chapters/concepts/get?chapterId=X`)
    ↓
ConceptList renders
🔹 C. Mark Concept Completed
Checkbox click
    ↓
markConceptCompleted(userId, conceptId)
    ↓
localStorage updated
    ↓
storage event triggers
    ↓
useEffect reloads progress
    ↓
UI updates
🔹 D. Watch Video
Thumbnail click
    ↓
markConceptAccessed(userId, conceptId)
    ↓
localStorage updated (lastAccessedAt)
    ↓
Video modal opens
🔹 E. Take Quiz
Click "Take Quiz"
    ↓
GET /api/quiz/get?conceptId=X
    ↓
QuizAttemptPanel renders
    ↓
(After attempt → likely stored locally, not DB)
🔹 F. Continue Learning (Dashboard)
useContentData()
    ↓
getProgressForUser()
    ↓
getContinueLearning()
    ↓
Shows last accessed OR next incomplete concept
🔹 G. Admin → Add Chapter
AddChapterDialog submit
    ↓
POST /api/chapters/add
    ↓
DB insert
    ↓
UI refresh (manual / implicit)
🔹 H. Admin → Edit Chapter
Edit input → Enter
    ↓
PUT /api/chapters/{chapterId}
    ↓
DB update
🔹 I. Admin → Delete Chapter
Click delete
    ↓
DELETE /api/chapters/{chapterId}
    ↓
DB delete
🔹 J. Admin → Add Concept
AddConceptDialog
    ↓
POST /api/chapters/concepts/add
    ↓
DB insert
🔹 K. Admin → Edit Concept
Edit concept
    ↓
PUT /api/chapters/concepts/{conceptId}
    ↓
DB update
🔹 L. Admin → Delete Concept
Click delete
    ↓
DELETE /api/chapters/concepts/{conceptId}
🔹 M. Profile Completion Flow
User logs in
    ↓
POST /api/user/profile
    ↓
DB → profileCompleted = true
    ↓
GET /api/user/check-profile
    ↓
Frontend decides access
